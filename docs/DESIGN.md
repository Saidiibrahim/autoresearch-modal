# Design Guide

## Goal

Design work in this repo is about agent legibility, not UI polish. Favor structures that make runtime intent, constraints, and operational paths easy to discover.

## Guardrails

- Keep the repo narrow and single-purpose.
- Put enduring design intent in `docs/design-docs/`, not in `AGENTS.md`.
- Prefer tables, maps, and exact command references over long narrative prose.
- When design intent changes, update the related product spec, architecture doc, and runbook in the same change.
